# NeonJudge Database Outline

Planned tables:

- users
- contests
- problems
- contest_problems
- submissions
- languages
- leaderboard_entries
- practice_records
- rating_history
- admin_audit_logs

Planned database features:

- constraints for data integrity
- triggers for leaderboard updates
- procedures/functions for reports and rating updates
- views for public leaderboard and dashboard summaries
- indexes for contest, submission, and problem search queries
